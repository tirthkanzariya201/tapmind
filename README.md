# TapMind Documentation

This repository is the **GitBook knowledge base** for the [TapMind](https://github.com/tirthkanzariya201/tapmind) platform. It is the single source of truth for how TapMind works, why it exists, and how its components fit together.

Documentation is written for developers, project managers, product managers, and clients. Every page follows a consistent structure so readers can quickly find what they need and understand both business context and technical behavior.

## Purpose

- Provide a clear, modular reference for the TapMind platform
- Explain business logic before implementation details
- Support onboarding, planning, and troubleshooting across teams
- Keep documentation independent so individual sections can be updated without rewriting the whole knowledge base

## Getting Started

1. Read [DOCUMENTATION_RULES.md](./docs/DOCUMENTATION_RULES.md) before contributing or editing any page.
2. Start with [01-Introduction.md](./docs/01-Introduction.md) for a platform overview.
3. Follow the numbered pages in order for a guided tour, or jump to the topic you need.

## Contributing

All documentation pages use the same section structure: Overview, Why It Exists, How It Works, Business Benefit, Failure Scenarios, and Related Components. See [DOCUMENTATION_RULES.md](./docs/DOCUMENTATION_RULES.md) for full guidelines.
