# Architecture

## Section Overview

This section explains how TapMind is structured at a system level. It covers the major components, how they connect, and how the platform fits into the broader ad monetization workflow.

Use this section after Getting Started when you need to understand the big picture before reading ad serving or reporting details.

## Who Should Read This Section

- **Product Managers** mapping platform capabilities to client needs
- **Project Managers** planning cross-team dependencies
- **Client Stakeholders** who want a clear view of how the system is organized
- **Support Teams** triaging issues that span multiple components
- **Developers** preparing for integration or debugging
